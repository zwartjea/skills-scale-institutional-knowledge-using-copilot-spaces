# Roles & Personas — Project Setup Checklist

Use this checklist when assembling a project team or updating project docs to ensure role clarity and accountability.

- [ ] Project One-pager lists proposed team roles and assigned owners
- [ ] For each role, document:
  - [ ] Role owner (name & contact)
  - [ ] Role summary (1–2 sentences)
  - [ ] Key responsibilities (3–6 bullet points)
  - [ ] Decision authority / approvals owned by role
  - [ ] Escalation path (who to contact next)
  - [ ] Expected deliverables and cadence (e.g., weekly report, test plan)
  - [ ] Onboarding artifacts (links to runbooks, design guidelines, test plan)
- [ ] Add interactions / handoffs (who depends on this role and when)
- [ ] Confirm acceptance criteria owner(s) for feature/story-level sign-off
- [ ] Add role definitions to docs/octoacme-roles-and-personas.md (or reference them)
- [ ] Share and review role assignments with stakeholders and delivery team (PM + PdM + Leads)
- [ ] Mark the project README with role definitions and link to relevant artifacts
